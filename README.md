## 📋 Обзор проекта
Разработка интеллектуального виртуального ассистента, предназначенного для автоматизации первичной коммуникации с клиентами в сфере интерьерного дизайна и ремонтных работ. Ассистент выступает в роли первого контактного лица: квалифицирует лиды, отвечает на специфические вопросы о услугах и материалах, а также автоматически бронирует встречи с дизайнерами в реальном времени.

**Основная цель:** Снижение нагрузки на менеджеров, исключение потери заявок в нерабочее время и ускорение процесса записи клиента на консультацию.

---

## ⚙️ Ключевой функционал

### 1. Осмысленное ведение диалога (Natural Language Understanding)
Ассистент поддерживает контекстный диалог, понимая не только прямые вопросы, но и скрытые намерения клиента.
*   **Контекстная память:** Помнит детали предыдущих реплик в рамках одной сессии (тип объекта, бюджет, сроки).
*   **Эмпатия и тон:** Общение выстроено в дружелюбном, профессиональном стиле, соответствующем премиум-сегменту услуг дизайна.
*   **Сценарии:** Умеет плавно переводить разговор от общих вопросов к квалификации лида и предложению встречи.

### 2. Интеллектуальная работа с базой знаний (RAG)
Система мгновенно находит точные ответы в внутренней документации компании, исключая галлюцинации и предоставление неверной информации.
*   **Источники данных:** Прайс-листы, этапы работ, примеры портфолио, информация о гарантиях, FAQ по материалам и срокам.
*   **Точность ответов:** Нейросеть формулирует ответы на основе фактов из базы, а не общих знаний из интернета.

### 3. Автоматизация записи встреч (Google Calendar Integration)
Полный цикл бронирования без участия человека.
*   **Проверка доступности:** Ассистент анализирует свободные слоты в расписании дизайнеров/менеджеров.
*   **Бронирование:** При согласии клиента создает событие в **Google Календаре**.
*   **Уведомления:** Формирует подтверждение встречи для клиента (дата, время, формат: онлайн/офис).

## 🛠️ Технологический стек и инструменты

Для реализации проекта был выбран современный стек инструментов, обеспечивающий баланс между качеством генерации текста и простотой интеграции.

| Компонент | Инструмент | Роль в проекте |
| :--- | :--- | :--- |
| **LLM (Ядровая модель)** | **Qwen** | Использована для разработки сложного системного промпта, настройки тональности (Tone of Voice) и тестирования логики работы с базой знаний перед внедрением. Обеспечивает высокое качество понимания русского языка и логики рассуждений. |
| **Платформа ассистента** | **Suvvy.ai** | Основная платформа для сборки бота. Обеспечивает интерфейс диалога, подключение базы знаний, управление сценариями и интеграцию с внешними API. |
| **Календарь** | **Google Calendar API** | Синхронизация расписания команды и автоматическое создание событий. |
| **База знаний** | **Векторная база (внутри Suvvy)** | Хранение и индексация документов компании для быстрого поиска релевантных фрагментов. |

## 🔄 Архитектура решения

1.  **Подготовка данных (Qwen):**
    *   Структурирование сырых данных компании (документы, чаты продаж).
    *   Генерация оптимального системного промпта с помощью Qwen, задающего роль "Опытного консультанта студии дизайна".
    *   Создание тестовых кейсов (диалогов) для валидации качества ответов.

2.  **Настройка платформы (Suvvy.ai):**
    *   Загрузка обработанной базы знаний в систему.
    *   Настройка параметров генерации (температура, максимальная длина ответа).
    *   Создание сценариев ветвления диалога (квалификация -> ответ на вопрос -> предложение встречи).

3.  **Интеграция:**
    *   Подключение модуля Google Calendar через нативную интеграцию Suvvy или webhook.
    *   Настройка триггеров: если клиент называет удобное время -> проверка слота -> создание события.

## 💡 Бизнес-ценность

*   **Экономия времени:** Менеджеры тратят до 40% меньше времени на рутинные ответы и согласование времени встреч.
*   **Работа 24/7:** Клиенты могут получить консультацию и записаться на встречу в любое время суток, включая выходные.
*   **Рост конверсии:** Мгновенная реакция на заявку повышает вероятность того, что клиент дойдет до реальной встречи.
*   **Стандартизация:** Все клиенты получают выверенную информацию согласно актуальной базе знаний, исключая человеческий фактор и ошибки новичков.

## 🚀 Результаты внедрения

*   Успешно настроен диалог, который клиенты часто принимают за живого оператора.
*   Реализована бесшовная интеграция с Google Календарем: встречи создаются автоматически с указанием всех необходимых деталей.
*   База знаний покрывает 95% типовых вопросов клиентов по ремонту и дизайну.
